# Style Guide

Contains data formatting conventions for the database that will be standardized
throughout **`ca_biositing.datamodels`** and **`ca_biositing.pipeline`**.

## Text Conventions

- All column names should be **lowercase** and in **snake case**. For example:
  - TARGET_FID -> target_fid
  - Join Count -> join_count

## Date Conventions

- All dates must be in **?????** format. For example:
  - 2024 Jan 28
- Do times matter??

## Geospatial Data

- Latitude and longitude should be stored as floats in two separate columns.
